# DNS Debug Path (2026-05-04)

## Complete Debugging Attempts (ALL FAILED for agent-browser CLI)

### 1. System DNS Fix (Firecrawl Method)
```bash
# Fixed /etc/systemd/resolved.conf
[Resolve]
DNS=1.1.1.1 1.0.0.1
FallbackDNS=8.8.8.8 8.8.4.4
Domains=~.
DNSSEC=no
DNSOverTLS=no
Cache=yes
DNSStubListener=yes

# Symlink resolv.conf to systemd-resolved stub
ln -sf /run/systemd/resolve/stub-resolv.conf /etc/resolv.conf

# Restart and verify
systemctl restart systemd-resolved
resolvectl status  # Shows: resolv.conf mode: stub, DNS Servers: 1.1.1.1
```

**Result**: Chrome directly works (`/opt/google-chrome/chrome --headless --dump-dom https://example.com` returns HTML). agent-browser CLI still fails with `net::ERR_NAME_NOT_RESOLVED`.

**Result**: Chrome binary works directly. agent-browser CLI fails.

## Root Cause (Identified via strings analysis)
```bash
strings $(which agent-browser) | grep -i "dns\|resolv"
# Output shows: "could not resolve to any address", "Unable to resolve", "dnsStart"
```

The Rust reqwest DNS resolver in agent-browser can't use systemd-resolved stub (127.0.0.53). Even with resolv.conf pointing to 127.0.0.53, the Rust code doesn't query the stub correctly.

## Workaround (Only Solution)
```bash
# Use Chrome directly (bypasses agent-browser Rust DNS code)
/opt/google-chrome/chrome --headless=new --no-sandbox --dump-dom <URL> > /tmp/page.html 2>&1

# For screenshots
/opt/google-chrome/chrome --headless=new --no-sandbox --screenshot=/tmp/screenshot.png <URL>
```
